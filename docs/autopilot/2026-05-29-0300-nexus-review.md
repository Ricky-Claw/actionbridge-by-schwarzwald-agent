### Token Shown-Once Semantics

- `createdSetupLink` state is populated **only** from the `POST` response (`body.setupLink`).

- `refresh()` / `loadSetupLinks()` calls `GET /api/actionbridge/setup-links`, which returns **no token field**.
- The list view (`setupLinks.map(...)`) cannot accidentally render a token because the GET API never returns it.
- The UI copy explicitly warns: _"After refresh/listing, only link id, status, methods, and expiry remain visible — not the raw token."_
