# Nexus Review — Webhook Secret Rotation Receiver-Smoke Gate — 2026-05-27 03:00 Europe/Berlin

## Decision
**GO** for this connector-only pilot hardening slice.

The change adds a UI-side receiver smoke-test evidence checkpoint before apply is enabled, updates the userflow smoke assertion, and keeps the existing dry-run-first/operator rotation boundary intact. No Critical/High Nexus blocker found in this delta.

## Architecture / product fit
- Fits the ActionBridge connector model: it hardens an existing operator control surface rather than creating a new execution path.
- Apply remains routed through the existing authenticated server API and explicit confirmation header; the UI only gates operator intent.
- The evidence field is local UI state and is not sent to the API. That is acceptable for the stated “local receiver-smoke evidence before enabling apply” requirement, but it is not a durable audit record.

## Boundary / safety review
- Connector-only boundary: **OK**. Touched docs, focused smoke test, and `ActionBridgeWebhookSecretRotationClient.tsx` only.
- No mock/fake behavior: **OK**. The smoke test checks rendered route content; the UI does not fake receiver success.
- Raw secret exposure: **OK for this delta**. No stored raw ref/secret is displayed. Placeholder explicitly says not to paste secrets. Operator-entered `nextSecretRef` remains an intentional input, not a leaked stored value.
- Sentinel posture: **OK with note**. The UI references receiver smoke as a required checkpoint before apply; enforcement remains UI-local, while the server still enforces connector auth, confirmation header, and existing policy marker.

## TypeScript / UI risk
- TypeScript risk is low: new state is a plain string and the disabled condition is simple.
- UX risk is low: apply is disabled until evidence has at least 12 trimmed characters; dry-run remains available without evidence.
- Watch item: length-only evidence validation is intentionally lightweight and can be bypassed by meaningless text. If this needs audit-grade enforcement, persist evidence server-side and include it in control audit metadata after redaction.
